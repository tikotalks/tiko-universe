# Tiko Ideology

Tiko is not a suite of productivity tools. It is a universe of gentle, focused tools for children and caregivers.

## The user model

The primary user may be a child who cannot or should not handle account setup. The caregiver may configure, recover, or transfer, but the first experience must not require adult account work.

## Identity philosophy

A Tiko user starts as a device user. That is a real user, not a temporary failure state. Adding an email makes that user recoverable. Magic-link verification links devices without introducing passwords.

## Interface philosophy

- Big controls.
- Few choices per screen.
- Visual consistency across apps.
- No hidden critical controls.
- Avoid scroll-dependent child interaction screens when possible.
- Parent/caregiver flows are available but not dominant.

## Platform philosophy

The platform exists to make apps simpler. If an app has to know how identity, storage, translation releases, or media authorization works internally, the platform is leaking.
