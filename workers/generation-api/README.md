# generation-api

Cloudflare Worker for generated assets. P0 supports text-to-speech; future sentence, image, and async job contracts should live here instead of creating one worker per provider.

## P0 routes

- `POST /v1/generation/tts`
  - Request: `{ text, language, provider?, voice?, model?, speed?, pitch? }`
  - Response: `{ data: { id, audioUrl, contentType, fileSizeBytes, generatedAt, provider, language, voice, model }, meta: { cached, schemaVersion } }`
  - Errors use `{ error: { code, message, field? } }`.
- `GET /v1/generation/audio/{id}`
  - Streams the generated R2 object after validating the D1 metadata row and safe object key.

## Storage

- D1 binding `GENERATION_DB` owns generation metadata (`generated_audio`).
- R2 binding `GENERATED_MEDIA_BUCKET` owns generated bytes.
- KV is intentionally not used as source of truth; future KV keys must be rebuildable caches only.

## Speech ownership

`workers/tts-api` is an Atlas-only adapter for `/generate`. It does not own providers, audio bytes, or generated-audio metadata.
